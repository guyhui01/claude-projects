## User Story
En tant qu'**administrateur Company-Test**,
Je veux **valider ou rejeter une demande de compte B2B et notifier le client**,
Afin de **contrôler l'accès à la plateforme**.

## Critères d'acceptation

**Scénario 1 — Validation du compte**
```gherkin
Given une demande de compte est en attente dans le back-office
When  je clique sur "Valider"
Then  le compte passe au statut "Actif"
And   le client reçoit un email "Votre compte est activé — vous pouvez vous connecter"
```

**Scénario 2 — Rejet avec motif**
```gherkin
Given je rejette une demande en saisissant un motif
When  je confirme le rejet
Then  le compte passe au statut "Refusé"
And   le client reçoit un email de refus avec le motif indiqué
```

**Scénario 3 — Notification admin**
```gherkin
Given un nouvel acheteur soumet une demande de compte
When  la demande est enregistrée
Then  l'administrateur Company-Test reçoit un email de notification
And   la demande apparaît dans la liste "En attente" du back-office
```
